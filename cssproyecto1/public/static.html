Static Animations

Static Animations, as the name implies, don't move. They can do a number of things.

A simple example is a box (yes, box fans! You can finally see one!) that changes color over time. Lets see it:

Cool, Right? Now, here's the code behind the scenes!

}
#example1 {
width:100px;
height:100px;
-webkit-animation: example1 10s;
animation: example1 10s;}
@-webkit-keyframes example1 {
from {background:lightslategrey}
to {background:black}
}
}

Now, lets break it down:

The name of the box is #example1. Thus, the styles below it will correspond to that object.

The height and width are set below it, each being 100px

The next two lines indicate that the box is to have an animation. The duration of the animation (10s) is also present. Use both lines, depending on what browser you have.

Then, the animation code begins. For this simple example, it goes from a lightlategrey background to a black one.

Now here's another example, which comes out with a similar product:

#example2 {
width:100px;
height:100px;
background-color: red;
-webkit-animation: example2 10s;}
@-webkit-keyframes example2 {
0% {background:red;}
25% {background:blue;}
50% {background:green;}
100% {background:black;}
}
